# Artifact Factory Pipeline Model

## Purpose

The Chummer media stack should behave like a repeatable artifact factory, not a collection of one-off vendor experiments.

Core rule:

> Approved source packets enter. Receipt-backed artifacts leave. Media tools never become truth.

## Standard pipeline

```text
Approved source packet
  -> script, article, digest, or briefing draft
  -> fact, spoiler, privacy, and publication review
  -> vidBoard presenter video where useful
  -> MarkupGo packet or bulletin
  -> PeekShot card or visual proof
  -> Soundmadeseen / Unmixr audio or subtitle support where useful
  -> Taja clips and social snippets after approval
  -> Signitic banner when passive campaign amplification is approved
  -> Emailit digest or lifecycle mail where appropriate
  -> ProductLift / changelog / guide link
  -> ClickRank / Katteb visibility review
```

## Repeatable outputs

- BLACK LEDGER city update
- KARMA FORGE house-rule explainer
- Community Hub open-run announcement
- mission briefing reel
- campaign primer
- season recap
- creator module trailer
- support fix-confirmation clip
- public guide article package

## Required receipt

Every artifact factory run must produce:

- source packet ref
- media recipe id
- source hash
- tool route and adapter version
- approval receipt
- render receipt
- publication ref
- takedown owner
- retention policy

## Boundaries

The artifact factory must not:

- invent lore or rules
- publish faction secrets by accident
- convert private support state into public claims
- treat video/social metrics as product truth
- skip accessibility, captions, and locale review where public
