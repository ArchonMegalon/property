# BLACK LEDGER Season Operator Playbook

## Purpose

This file defines the human operations posture for BLACK LEDGER seasons.

BLACK LEDGER needs operators, not only code. The season loop must make authority, review, publication, closeout, and escalation visible.

## Roles

- Product Governor
- world operator
- season operator
- faction-seat coordinator
- intel curator
- GM coordinator
- media publisher
- support/privacy reviewer

## Weekly loop

```text
collect intel and resolution reports
  -> review faction operations
  -> validate job seeds and open runs
  -> approve or reject world tick candidates
  -> produce map/newsreel/digest/source packets
  -> publish first-party truth
  -> amplify through Signitic, Emailit, Taja, ProductLift, and guide pages
  -> review journey proof and support side effects
```

## Required tools

- Hub for canonical state
- Teable for workbench projections
- NextStep for SOPs
- ApproveThis for external approvals
- MarkupGo, PeekShot, vidBoard, and Taja for media outputs
- Signitic for passive managed-account campaigns
- Emailit for digests and closure messages

## Never automate

- faction victory
- leaderboard rank
- player discipline
- campaign canon publication
- public faction secrets
- support closure

Those require typed source, authority, and review.
