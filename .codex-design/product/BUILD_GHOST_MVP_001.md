# Build Ghost MVP 001

Build Ghost is the guided explain lane for users who need help understanding why a build is valid, blocked, or strategically weak without leaving the workbench.

The MVP posture is intentionally guided, not decorative:
- it must explain why a recommendation exists,
- it must point back to the exact workbench surface that can act on the advice,
- and it must stay grounded in authored rules, sources, and current runner state.

Build Ghost is not a replacement for dense expert controls.
It is the explain-and-recover companion that helps a player or GM keep momentum when the workbench exposes a surprising rule interaction, missing prerequisite, or source conflict.

The first-pass Build Ghost contract is:
- explain the current blockage or recommendation in plain language,
- cite the governing rule, source, or build constraint,
- provide a guided next action that returns the user to the exact surface that resolves the issue,
- and never invent hidden math, fake certainty, or unsupported parity claims.
