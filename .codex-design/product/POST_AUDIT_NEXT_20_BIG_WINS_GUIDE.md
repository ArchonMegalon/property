# Chummer post-audit next 20 big wins

## Framing

This is a **post-audit dev guide**, not a replacement for the canonical
`NEXT_20_BIG_WINS_REGISTRY.yaml`.

The live design repo now states the prior wave is closed. The next sequence is therefore:

1. close the still-open milestones the repos already admit,
2. graduate campaign and control package families from "real enough" to boringly complete,
3. deepen campaign/product cohesion,
4. widen the moat around Build / Explain / Publish / Improve.

## Ordering rule

Close open current-wave truth first.
Then make campaign and workspace center of gravity coherent across Hub, UI, mobile, and Registry.
Then widen publication, community, and guided help into visible, repeatable product pillars.
