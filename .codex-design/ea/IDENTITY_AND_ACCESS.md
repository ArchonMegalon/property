# Identity And Access

Identity should feel boring.
The product must make it obvious how a person starts and how they come back tomorrow.

## Create

Default creation path:

- enter email
- create workspace
- continue in the current session

Google is workspace data setup, not the primary app identity method.

## Return

Supported return paths:

- current session
- secure email link
- workspace invite link
- managed SSO or access gateway

Do not imply a generic login screen if the real return method is a secure link or a managed access layer.

## Sign-in page rules

The sign-in page must:

- tell returning users which access paths are valid
- tell new users to start at registration
- describe the current-session path explicitly
- keep Google framed as data connection, not identity shortcut

## Approval and invite links

Approval links and invites are part of the identity story.
They should reopen the right workspace directly and explain what happens next.
