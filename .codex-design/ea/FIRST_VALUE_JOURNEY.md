# First Value Journey

## Goal

Get one person from signup to a useful morning brief and one cleared review item without teaching the entire office model.

## Sequence

1. Enter email and create the workspace.
2. Pick the workspace shape.
3. Connect Google for workspace data.
4. Build the first morning brief.
5. Review one queue item and one follow-up.

## Identity rule

The return path should feel boring:

- create with email
- return with an email link, invite link, or current session
- treat Google as workspace data setup, not the primary app identity story

## Success conditions

- the user sees a morning brief worth reading
- at least one draft or decision is reviewable
- at least one commitment or follow-up stays visible
- the user can tell where queue work ends and commitment tracking begins
- the user understands that nothing sends without review
