# Chummer Grounded Explain Narration Packs

This package lands the EA-owned slice for milestone `145`:

- `CHUMMER_EXPLAIN_NARRATION_PACKET_PACK.yaml` defines the compile contract for packet-grounded narration and bounded follow-up copy.
- `GROUNDED_FOLLOW_UP_SPECIMENS.yaml` captures representative desktop and mobile specimen packs for `why`, `why_not`, `what_if`, `what_changed`, and `source_anchor`, all built strictly from packet truth.
- `SUCCESSOR_HANDOFF_CLOSEOUT.yaml` records the active package boundary, canonical authority, and repeat-prevention posture while sibling repos finish their packet emitters and explain surfaces.
- `skills/chummer_grounded_explain_narration/SKILL.md` gives workers one bounded compile workflow that stays subordinate to packet and counterfactual truth.

The package is intentionally text-first and fail-closed. If packet identity, stale-state posture, privacy class, source anchors, or bounded counterfactual support are missing, the compile must emit an unavailable or inspect-the-packet result instead of invented narration.
