# PropertyQuarry Tone Guide

PropertyQuarry should sound premium, calm, analytical, and direct.

Avoid:

```text
EA on customer surfaces
OODA on customer surfaces
raw backend codes
raw URLs in message text
AI magic language
broker hype
unsupported certainty
```

Use:

```text
Confirmed
Likely
Unclear
Missing
Needs owner review
Official-source backed
Provider-only
User-reported
```

Examples:

| Bad | Good |
| --- | --- |
| Candidate generated. | Property page is ready. |
| premium_dossier_render_failed | The premium PDF did not pass quality checks. |
| No result. | No listing met this brief. Try relaxing one constraint. |
| Open: https://... | Open the titled review button. |
| OODA summary | Decision summary |
| Artifact receipts | Send checklist |

Failure UX must always include:

```text
human message
operator detail
retry action
fallback action
receipt
```
